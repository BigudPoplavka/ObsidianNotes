В гомогенных вычислениях для выполнения приложения используется один или несколько процессоров одной и той же архитектуры. Вместо этого гетерогенные вычисления используют набор процессорных архитектур для выполнения приложения, применяя задачи к наиболее подходящим архитектурам, чем обеспечивают в результате повышение производительности.

В настоящее время типичный гетерогенный вычислительный узел состоит из двух многоядерных **CPU** сокетов и двух или более многоядерных **GPU**. **GPU** в данный момент является не отдельной платформой, а сопроцессором **CPU**. Поэтому GPU должен работать совместно с хостом на базе **CPU** через шину **PCI-Express.** Именно поэтому, с точки зрения вычислений на **GPU**, **CPU** называется хостом, а **GPU** – устройством.

Гетерогенное приложение состоит из двух частей:
- код хоста - выполняется на **CPU**
- код устройства - на **GPU**

Есть две важные особенности, которые описывают возможности **GPU**:
- число ядер **CUDA** - дает метрику **пиковой производительности**
- размер памяти - дает метрику **пропускной способности памяти**
