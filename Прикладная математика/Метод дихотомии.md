[[Метод золотого сечения]]
[[Математическая функция]]

Применяется для оптимизации управления одномерной функции `J(U)`.  Метод требует только непрерывности функции. Это метод **двойного деления**, двумя точками делим отрезок. 

Точки будем брать на том расстоянии, которое соответствует точности `delta` нахождения **оптимума**. 

![[Pasted image 20231010134502.png]]

- ## Алгоритм
	1. Находим точки `u1 = (a + b - delta) / 2, u2 = (a + b + delta) / 2`, то есть от середины отрезка `[a; b]` смещаемся на дельту в обе стороны 
	2. Вычисляем **значение критерия качества оптимизации** `J(u1) и J(u2)`
	3. Если `J(u1) <= J(u2)`, то `b = u2` - отбрасываем от `u2`, а не `u1`, так как **хоть** условие `J(u1) <= J(u2)` может и **выполнится**, **но оптимум будет между** точками и таким образом его можно случайно выкинуть.
	4. Повторяем для нового отрезка, пока отрезок не станет `< delta`.
	5. Если отрезок стал по величине ниже заданной `delta`, то берем за решение последние `(a + b) / 2`

- ## Особенности
	1. При необходимости высокой точности, то дробление отрезка будет очень длительным по времени
	2. Улучшением этого метода является метод взятия точек не просто как на расстоянии `delta`, а брать **золотые сечения**

- ## Преимущества
	1. Прост
	2. Хорош только для **одномерных** функций
	3. Гарантирует результат

- ## Недостатки
	1. Не эффективен по скорости за счет малости дробления
